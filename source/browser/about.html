FORTH-like

What do we mean by FORTH? And what does it mean to be like FORTH?

FORTH is a very interesting programming language invented, or some say "discovered", by Charles "Chuck" Moore in the late 1960s, and is given the birth year of 1970. It was originally used for controlling telescopes, and has found a solid home for embeded programming. This is due to its rather spartan, syntax-free design. This allows language core to be implemented in a few dozen or hundred lines of assembly language. This feature has made it very popuplar for microcontrollers and generally for embedded applications.

As FORTH has been used over the decades, there has arisen a rather large standard for FORTH, and there are many implementations. Chuck Moore, the inventor or "discoverer" of FORTH, however, describes it as a "personal programming language". This is due to its minimal and easily extensible nature. His approach is to start very small, and only add functionality required for the problem at hand.

So FORTH has these two faces - standardized and large (some would say bloated) and personal, minimal, and built to-purpose.

This project, Symbolic FORTH, tries to take Chuck's approach. In fact, I would add one additional attribute - adapted to the platform it is running on.

Chuck and most FORTHers work with small microcontrollers and thus need to deal with assembly language as the base implementation. This means that much of the base implementation is concerned with the characteristics of the MC, with the board it is integrated into. However, when implementing on top of Javascript and a web browser, we do not have those same concerns. Rather, the basis for our FORTH is Javascript itself. Thus, we can exploit many charactersicts of JS, like garbage collection for memory management, JS datatypes, and JS programming.
