Types

Traditional FORTH does not support value types in the modern sense. There are integers and words. (A word being equivalent to a function).

Since the FORTH system recognizes no types, there is also type checking. Of course, one may implement such things in one's own words - but the language does not support type checking. More complex types like doubles, strings, arrays, and so forth do exist, but are typically created and used with bespoke "words" (like functions).

Symbolic FORTH does supoprt types, in several ways.

  • All values carry a type specifier. Currently this is implemented as a Javascript array, in which the first element is a type id (string), and the second the value.
  • Internally some stack operations check the requested type, and throw an error if the type is not as expected. E.g. forth.pop_number() will pop a value off of the stack, check that the type is 'number', throw an error if it is not, and return the numeric value if it is.
  • Some generic or partially generic words are supported; however most are type-specific. For instance, eq is a generic equality word, whereas = is just for numbers and S= just for strings.
  • Some standard features have been extended to support type-specific features. For example, the to store a value from the stack as a local, the L:> word may be used; to store all of the values of an array from the stack into a set of local variables, the L:>[ .. ] word may be used.

Built-In Types

Symbolic FORTH is based on browser Javascript so it generall supports all basic types available underneath. This allows Symbolic FORTH to be more easily used to program directly against the browser DOM. Yes, it might be fun one day to fork this FORTH and limit it to a more basic type and build from there, but not yet, perhaps never.

This FORTH provides several data types beyond just the integer. In fact, the basic type is the number, not integer, as this is what Javascript offers.

All Symbolic Forth values are stored with their typing intact. This allows us to use polymorphic words when appropriate. For instance, the output word . will print the canonical form of any value provided to it from the Data Stack. However, other types of words like comparisons (equality, less than, etc.) are not polymorphic because the way one compares values of different types are too different. In such cases a generic approach can lead to confusion.

number
A Javascript number - either integer or decimal form accepted, positive or negative. Examples: 123, 1.23, 0, -456
string
A Javascript string, provided in the classic FORTH style, using the double-quote as a word to start reading a string from the program stream. A string ends with the character ". Examples: " a string"
boolean
A Javascript boolean, provided as a raw token of either true or false. Examples: true, false
null
Nothing much to see here other than the fact that the null value is represented as the null type - no value is actually needed.
symbol
This type does not have a Javascript equivalent; it is simply stored as a string. A symbol is basically a single-word string value, with the string stored in a separate symbol table. Within FORTH code it is provided by using a ' prefix. [If FORTH encounters a token which is not a word, number, or boolean, it is interpreted as a symbol. Examples: foo VAR bar, foo .]
array
An array is simply a sequence of any number of FORTH values. We do not currently support words in an array. Arrays may be created, any member accessed, and any member set. We may also get the length of an array.
map
A map is essentially a Javascript Object, with string keys and any FORTH value as the value.
