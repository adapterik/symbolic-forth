UI

A focus of Symbolic FORTH is to provide both basic and sophisticated user interfaces and a user interface toolkit for building bespoke user interfaces.

The basic user interface consists of an output area, a console area, and an input area.

The output area is where any ordinary "print" statements are displayed, as well as normal program feedback. Abnormal program feedback is displayed in the console area. User input is provided in an input area.

This is very simple, perhaps overly so. It is also important that this is currently just a basic starting point. One can easily imagine a classic REPL which combines all three i/o needs.

FORTH Editor

Although any editor can be used for FORTH code, the programming support for editors does not suit FORTH programming adequately. For instance, as words, variables, constants, and other runtime resources are defined, they should be recognized by the editor from that point forward, though not previously.

It would also be nice to be able to simply select code and execute it.

IDE

The programming tool is the playground for a FORTH IDE. FORTH, due to the simple and understandable virtual machine, is very suitable to an IDE that exposes the internal data, such as words, variables, contstants, arrays, objects, and the like.
